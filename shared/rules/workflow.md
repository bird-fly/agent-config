# Skill 使用总则

## 一句话原则

**"不确定用什么？先问 `/ask-matt`"**

---

## 技能选择流程

1. **不确定** → `/ask-matt`（技能路由器）
2. **明确场景** → 看下表选技能
3. **无合适技能** → 直接干活

---

## 快速选技能

| 我要做什么 | 用什么 |
|-----------|--------|
| 不知道用啥 | `/ask-matt` |
| 需求不清楚 | `/grill-me` |
| 调研技术 | `/research` |
| 复杂 bug | `/diagnose` |
| 优化架构 | `/improve-codebase-architecture` |
| 代码审查 | `/requesting-code-review` |
| 写测试 | `/tdd` |
| 做漂亮页面 | `/design-taste-frontend` |

**其他技能：** `/prototype` `/teach` `/to-issues` `/to-prd` `/handoff` `/codebase-design`

---

## 记住三点

1. **简单任务不用技能** - 直接干，别搞复杂
2. **一次只用 1-2 个技能** - 别堆砌
3. **用户说"快速"时听话** - 别自作主张走完整流程
