# 全局行为准则

## 双模式智能切换

### 简单模式（默认）
- 简短直接，minimal-code，对付 bug/小改/日常
- 触发词：`简单`/`快速`/`小改`/`修复`/`bug`/`优化`

### 复杂模式（技能驱动）
- 关键环节用 Matt Pocock 技能
- 触发词：`架构`/`规划`/`重构`/`设计`/`新功能`/`复杂`

**常用技能速查：**
- 不确定用什么 → `/ask-matt`
- 需求澄清 → `/grill-me` 或 `/grill-with-docs`
- 调研新技术 → `/research`
- 排查 bug → `/diagnose`
- 架构优化 → `/improve-codebase-architecture`
- 代码审查 → `/requesting-code-review`
- 测试驱动 → `/tdd`
- 设计规范 → `/codebase-design`

## 语言与交互

**用户使用中文时：**
- ✅ 用中文回复（代码注释、变量名除外）
- ✅ 代码、命令、技术术语保持英文
- ✅ 文件名、路径、配置保持原样

**示例：**
```
用户：帮我优化这个函数
回复：我来帮你重构这个函数，提升性能...
代码：function optimizeQuery() { ... }  // 优化查询性能
```

## Karpathy 编码准则

- **先思考再编码**：不准默默做假设，模糊就提问，困惑立刻停下
- **简约至上**：只写最小可工作代码，不准搞没人要的抽象和灵活性
- **手术式修改**：只碰你要求的部分，不准顺便重构邻居代码
- **目标驱动执行**：先写成功标准，每一步都要可验证

**原则：**
```
可工作的简单代码 > 不工作的完美代码
明确的重复代码 > 抽象但难懂的代码
手术式小改动 > 全面的大重构
```

## Windows 中文兼容

- 文件统一使用 UTF-8 无 BOM
- Windows shell 输出中文异常时，先执行 `chcp 65001`
- 中文路径用引号：`"E:\项目\测试.md"`

## 规则优先级

1. 用户明确指令（最高）
2. 客户端专属规则（`clients/{client}/rules/`）
3. 共享规则（`shared/rules/`）
4. 技能内部规则（最低）
