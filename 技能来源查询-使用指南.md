# 🔍 技能来源查询 - 使用指南

> 解决"下载了很多技能，不知道来源"的问题

## ✨ 新功能

现在有 **两个强大的工具** 帮你识别技能来源和归属！

---

## 🚀 快速开始（3 种方式）

### 方式 1: 查看单个技能（最常用）

```powershell
# Node.js 工具（详细）
node scripts\analyze-skills.js --skill grill-me

# PowerShell 工具（快速）
.\scripts\skill-source.ps1 grill-me
```

### 方式 2: 交互模式（最方便）

```powershell
.\scripts\skill-source.ps1 -Interactive

# 然后输入技能名，随时查询
```

### 方式 3: 查看所有技能来源

```powershell
# 列表模式
.\scripts\skill-source.ps1 -All

# 按来源分组
.\scripts\skill-source.ps1 -BySource
```

---

## 📊 工具对比

| 功能 | Node.js 工具 | PowerShell 工具 |
|------|--------------|-----------------|
| 查看单个技能 | `--skill <name>` | `<name>` |
| 交互模式 | ❌ | ✅ `-Interactive` |
| 完整分析报告 | ✅ 默认 | ❌ |
| 按来源分组 | ✅ `--verbose` | ✅ `-BySource` |
| 列出所有技能 | ✅ 在报告中 | ✅ `-All` |
| 依赖关系检测 | ✅ | ❌ |
| 启用状态显示 | ✅ | ❌ |
| JSON 输出 | ✅ `--json` | ❌ |

**推荐：**
- 🎯 **日常快速查询** → PowerShell 交互模式
- 📊 **完整分析** → Node.js 工具
- 📝 **生成报告** → Node.js JSON 输出

## 💡 典型使用场景

### 场景 1: "这个技能是谁的？"

```powershell
# 最快方式
.\scripts\skill-source.ps1 to-issues

# 输出：

# 📝 描述: Break a plan, spec, or PRD into issues...

# 📦 来源: Matt Pocock
```

### 场景 2: "我想知道所有 Matt Pocock 的技能"

```powershell

```

### 场景 3: "这个技能是 Superpowers 的吗？"

```powershell
node scripts\analyze-skills.js --skill brainstorming

# 输出会显示：
# 📁 分类: superpowers
# 📦 来源: Superpowers Framework
```

### 场景 4: "连续查询多个技能"

```powershell
.\scripts\skill-source.ps1 -Interactive

# 然后：
输入: grill-me
输入: diagnose
输入: to-issues
输入: exit
```

## 🔍 来源检测规则

工具会自动检测以下特征：

| 来源 | 检测依据 |
|------|----------|
| **Matt Pocock** | 文档包含 "mattpocock" / "Matt Pocock" |
| **Superpowers** | 使用 `docs/superpowers/` 或 `~/.config/superpowers/` 路径 |
| **Multica** | 技能名以 `multica-` 开头，或文档包含 "multica" |
| **Anthropic** | 文档包含 "anthropic" 或 "Anthropic" |
| **GitHub 项目** | 文档包含 GitHub 仓库链接 |
| **未知来源** | 以上都不匹配 |

## ⚙️ 高级功能

### 自定义来源检测

如果需要识别更多来源，编辑 `scripts/analyze-skills.js` 中的 `detectSkillSource` 函数：

```javascript
// 检测技能来源和作者
function detectSkillSource(skillName, content, metadata) {
  const sources = [];
  
  // 添加你的自定义检测逻辑
  if (content && content.includes('your-custom-marker')) {
    sources.push('Your Custom Source');
  }
  
  // ...
}
```

### 批量操作

```powershell
# 批量查询多个技能
$skills = @('grill-me', 'diagnose', 'to-issues')
foreach ($skill in $skills) {
    .\scripts\skill-source.ps1 $skill
}

# 导出所有技能信息到 CSV
node scripts\analyze-skills.js --json | 
    ConvertFrom-Json | 
    Select-Object -ExpandProperty sources | 
    ConvertTo-Csv | 
    Out-File skills-export.csv
```

## 🆘 故障排查

### 问题: Node.js 命令不工作

```powershell
# 检查 Node.js 是否安装
node --version

# 确保在项目根目录
cd e:\Project\codexProject\agent-config
```

### 问题: PowerShell 脚本权限错误

```powershell
# 设置执行策略（以管理员运行）
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

### 问题: 找不到技能

```powershell
# 确认技能存在
Get-ChildItem shared\skills -Directory | Select-Object Name

# 检查技能名是否正确（区分大小写）
```
